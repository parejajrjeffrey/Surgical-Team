# 📜 RULE REGISTRY — Frontend UI/UX Engineering

38 frontend-specific decision rules, best practices, and anti-patterns.

| # | Rule | Category | Severity | Anti-Pattern (DO NOT) |
|---|---|---|---|---|
| R001 | All design values (colors, spacing, typography, shadows, radii) must come from design tokens — never hardcoded hex codes, px values, or font sizes | Design System | 🟥 CRITICAL | `color: #3B82F6; padding: 13px; font-size: 15px` — bypasses design system |
| R002 | Every `<img>` must have a meaningful `alt` attribute — describe image purpose; decorative images get `alt=""` with `role="presentation"` | Accessibility | 🟥 CRITICAL | `<img src="chart.png">` with no alt — screen reader says "image" |
| R003 | Keyboard navigation must work for all interactive elements — Tab order matches visual order; Enter/Space activates; Escape dismisses; custom widgets must implement ARIA keyboard patterns | Accessibility | 🟥 CRITICAL | `<div onClick={handleClick}>` — clickable with mouse, unreachable by keyboard |
| R004 | Bundle size must be monitored per route — no initial route chunk >100KB gzipped JS; use route-level code splitting; audit third-party libraries via bundlephobia.com | Performance | 🟥 CRITICAL | `import moment from 'moment'` — 72KB gzipped for date formatting; use `date-fns` or `Intl.DateTimeFormat` |
| R005 | Forms must have associated labels, error messages, and accessible validation — `<label htmlFor={id}>`, errors use `aria-describedby`, `aria-invalid="true"`, `aria-required="true"` | Accessibility | 🟥 CRITICAL | Placeholder-only form with no labels — disappears on focus, low contrast |
| R006 | State must be categorized: Server state→TanStack Query/SWR; URL state→useSearchParams; Form state→React Hook Form; UI state→useState; Global→Zustand/Jotai; never server data in global stores | State Management | 🟧 HIGH | `useState(null); useEffect(()=>fetch('/api/user'))` in 12 components |
| R007 | Animation must use GPU-accelerated properties only — `transform`, `opacity`; never animate `width`, `height`, `top`, `left`, `margin`, `padding` | Performance | 🟧 HIGH | `transition: width 0.3s ease` — 30fps vs 60fps with `transform: scaleX()` |
| R008 | `prefers-reduced-motion` must be respected globally — wrap animations in `@media (prefers-reduced-motion: no-preference)`; disable auto-playing videos/carousels when reduced | Accessibility | 🟥 CRITICAL | Auto-playing video background, parallax, animated charts with no disable |
| R009 | Color contrast must meet WCAG AA minimum — 4.5:1 normal text, 3:1 large text, 3:1 UI components; test with axe DevTools | Accessibility | 🟥 CRITICAL | `color: #9CA3AF; background: #FFFFFF; font-size: 14px` — 2.8:1 contrast |
| R010 | Images must have explicit `width` and `height` to prevent CLS — aspect ratio box reserved before load; WebP/AVIF with `<picture>` fallback | Performance | 🟧 HIGH | `<img src="hero.jpg">` with no dimensions — CLS = 0.4 |
| R011 | Components must be composable and props-type-safe — export TypeScript interface for props; destructure with defaults; never `any`-typed props | Architecture | 🟧 HIGH | `const Button = (props: any) => { … }` — no autocomplete, no type checking |
| R012 | CSS specificity must be managed — use Tailwind utilities or CSS Modules; never `!important` (except third-party override with comment); max specificity depth of 3 | CSS Architecture | 🟧 HIGH | `.page .content .card .title h2 span { color: red !important; }` |
| R013 | Secure against XSS — never use `dangerouslySetInnerHTML` without DOMPurify sanitization; validate `href` URLs — prevent `javascript:` protocol injection | Security | 🟥 CRITICAL | `<div dangerouslySetInnerHTML={{ __html: userInput }} />` with no sanitization |
| R014 | Touch targets must be ≥44×44px on mobile (WCAG 2.5.5) — add padding to small icons/buttons; no overlapping touch targets | Accessibility | 🟧 HIGH | `X` close button 16×16px with no padding |
| R015 | Font loading must not block rendering — `font-display: swap`, preload critical woff2, fallback font with `size-adjust` | Performance | 🟧 HIGH | `font-display: block` — 3-second invisible text |
| R016 | SSR must not leak client-only state — no `window`, `document`, `localStorage` during SSR; use `useEffect`, `typeof window !== 'undefined'`, or `dynamic(()=>import(…), { ssr: false })` | Architecture | 🟥 CRITICAL | `localStorage.getItem('theme')` at module top-level — crashes SSR |
| R017 | Loading, empty, error states must all be handled — loading (skeleton), empty (CTA), error (retry), success; error boundaries per route | UX | 🟧 HIGH | Component with only success path — blank on loading, cryptic on failure |
| R018 | React renders optimized based on profiler data — `useMemo`/`useCallback`/`React.memo` only when profiler shows measurable benefit (>1ms render) | Performance | 🟨 MEDIUM | `React.memo(({ onClick }) => {…})` — but `onClick` is inline, memo 100% useless |
| R019 | Semantic HTML must be used — `<header>`, `<main>`, `<nav>`, `<footer>`, heading hierarchy no skip, `<main>` on every page | Accessibility | 🟧 HIGH | `<div class="header"><div class="nav"><div class="article">` |
| R020 | Custom hooks must be extractable — >50 lines or used in >2 components → `hooks/use<Name>.ts`; name starts with `use`; never call hooks conditionally | Architecture | 🟨 MEDIUM | 200-line component with 4 inline `useEffect` and 3 inline `useState` |
| R021 | Forms must prevent double-submission — disable submit button until response; use `isSubmitting` or `isPending`; server-side idempotency key for critical forms | UX | 🟧 HIGH | Clicking "Place Order" 3 times creates 3 duplicate orders |
| R022 | All interactive components must have visible focus indicators — never `outline: none` without `:focus-visible` alternative; focus indicator ≥3:1 contrast | Accessibility | 🟥 CRITICAL | `*:focus { outline: none; }` in global CSS reset |
| R023 | URL must reflect application state — filters, pagination, sort, open modals in URL params/path; no hidden state unshareable via URL | UX | 🟧 HIGH | Complex search at `/search` — sharing gets friend to page 1 with no filters |
| R024 | Third-party scripts strategically — `<Script strategy="lazyOnload" />` or `async/defer`; audit quarterly; use Partytown for non-critical scripts | Performance | 🟧 HIGH | `<script src="analytics.js">` in `<head>` blocking render for 3 seconds |
| R025 | Error boundaries must wrap every page/section — at least one per route; fallback UI with retry; never let render error crash entire app to white screen | Architecture | 🟧 HIGH | No error boundary — `TypeError` in sidebar widget crashes entire SPA |
| R026 | CSS logical properties for i18n — `margin-inline-start` not `margin-left`, `padding-block` not `padding-top`; prepares for RTL languages zero refactoring | CSS Architecture | 🟨 MEDIUM | All CSS physical directions — RTL support requires rewriting every horizontal property |
| R027 | Typography modular scale with fluid sizing — `clamp(min, vw+rem, max)`; line-height 1.5 body, 1.3 headings; max-width 65ch; never <16px body (iOS zoom bug) | Design System | 🟨 MEDIUM | `font-size: 12px; line-height: 1.2` for body — unreadable |
| R028 | Conditional rendering handle all branches — if/else with explicit fallback; use `items.length === 0` not `!items` (0 is falsy!) | UX | 🟧 HIGH | `{items && <List />}` — items=[] shows empty list, items=0 shows nothing |
| R029 | TypeScript strict mode — `strict: true`; no `as any`; discriminated unions over optional properties; `satisfies` for config; `unknown` over `any` | Type System | 🟧 HIGH | `tsconfig.json` with `strict: false` and 47 `as any` casts |
| R030 | Design must work at 200% zoom (WCAG 1.4.4) — no fixed-height containers that clip; relative line-height; no horizontal scroll at 200% on 1280px | Accessibility | 🟧 HIGH | `height: 40px; overflow: hidden` on text — half the text invisible at 200% |
| R031 | Environment variables namespaced — `NEXT_PUBLIC_`/`VITE_` for client; never secrets to client; validate at build; `.env.example` documented | Security | 🟥 CRITICAL | `NEXT_PUBLIC_STRIPE_SECRET_KEY=sk_live_...` shipped to every browser |
| R032 | Skeleton screens match content layout — same dimensions/spacing; `aria-busy="true"`; transition smoothly; never generic rectangle/spinner | UX | 🟨 MEDIUM | `<Spinner />` centered — signals indefinite wait, skeletons signal progressive loading |
| R033 | Event handlers must be cleaned up — `addEventListener`→cleanup; `setInterval`→`clearInterval`; WebSocket→close; Observer→disconnect | Performance | 🟧 HIGH | `useEffect(()=>{window.addEventListener('resize', handler)})` no cleanup — 20 handlers after 20 navigations |
| R034 | Form validation on blur AND submit — inline feedback after first touch, not on pristine; full validation on submit; mode: `onBlur` or `onTouched` + `onSubmit` | UX | 🟨 MEDIUM | `onChange` validation: "Must be 8 chars" after first letter typed |
| R035 | Icons must be accessible — SVG: `aria-hidden="true"` decorative, `role="img"`+`<title>` meaningful; icon-only buttons need `aria-label`; never icon fonts | Accessibility | 🟧 HIGH | `<button><Icon name="trash"/></button>` — screen reader: "button" no label |
| R036 | Lazy loading below-fold only — `loading="lazy"` on below-fold `<img>`/`<iframe>`; `React.lazy`/`next/dynamic` for below-fold components; never lazy-load LCP image | Performance | 🟧 HIGH | `loading="lazy"` on hero LCP image — LCP increases 2 seconds |
| R037 | `<title>` descriptive and unique per page — `Page Name | App Name`; dynamic for parameterized pages; never just app name on every page | Accessibility/SEO | 🟧 HIGH | Every page: `<title>My App</title>` — 50 tabs indistinguishable |
| R038 | Progressive enhancement as default — core content works without JavaScript (SSR/SSG); JS enhances interactivity; test with JS disabled | Architecture | 🟨 MEDIUM | `<body>` is empty `<div id="root">` — blank white screen without JS |
